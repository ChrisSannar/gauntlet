# Flagship: CodeCardio (Weeks 8–11)

*Bring https://www.codecardio.org to MVP: make and edit drill courses with AI.*

Everything from weeks 1–7 assembles here — re-implemented against CodeCardio's
existing **Go** backend (React/TS front, Postgres). The language pivot is deliberate:
transferring AI-engineering patterns across languages is the senior signal
(ADR 0007). Deployment bar is **Tier 2** (ADR 0010): custom domain, working auth,
payment rail, error monitoring.

**MVP definition** (what "fully developed" means, per ADR 0008): a stranger can sign
up, generate an AI-built drill course on a topic, practice drills with automated
grading, return on later days to a spaced-repetition review queue, edit their course
via AI feedback, and hit a payment gate — with the whole thing monitored in
production.

## Week 8 Milestone — Re-entry & Hardening

*The codebase becomes trustworthy before it grows. Front-end (familiar React/TS)
carries momentum while Go skills come back online.*

1. **Flagship product brief** (`docs/BRIEF.md`): the MVP definition above translated
   into your own scoped feature list, with explicit non-goals.
2. **Full bug triage** (`docs/TRIAGE.md`): every known defect inventoried, reproduced,
   and prioritized P0–P3 with rationale.
3. **All P0 and P1 bugs fixed**, each with a regression test.
4. **Auth solidified**: signup, login, logout, session expiry, protected routes — all
   verified by tests.
5. **Error monitoring live** in production (Sentry-class), proven by a deliberate test
   error appearing in the dashboard.
6. **CI green**: build + full test suite runs on every push; failing CI blocks merge.
7. **Navigation/UX debt cleared**: no truncated nav, no dead links, empty states
   handled.

## Week 9 Milestone — The Forge, in Go

1. **AI course creation in-product**: topic → structured drill course (units →
   lessons → drills), schema-constrained, generated by the Go backend. Courses are
   persisted, browsable, and immediately practicable.
2. **Objectives + pedagogy rules ported**: measurable objectives per lesson,
   lesson-sizing rules enforced (Project 2's `PEDAGOGY.md` applied here).
3. **Content-quality eval suite ported** and running in CI against generated courses
   (harness language is your choice — hitting the Go API is fine — but it must run
   in CI with thresholds).
4. **LLM observability in Go**: model, tokens, cost, latency logged per call;
   provider-agnostic model configuration (ADR 0009).
5. **≥1 ADR** on the Go AI-service architecture.

## Week 10 Milestone — The Judge & The Pen

1. **Automated drill grading in-product** (Project 3): free-form answers judged with
   structured verdicts + abstain path; **judge-accuracy harness re-run** in the
   CodeCardio context with a committed report.
2. **Spaced repetition in-product**: FSRS state per user per drill in Postgres; review
   queue; mastery gating on unit advancement; scheduler tests with a test clock.
3. **Agentic course editing in-product** (Project 4): NL feedback → plan → scoped-tool
   edits → reviewable diff → apply/rollback, with audit log and edit budget.
4. **Scenario evals re-run** against CodeCardio's editor (≥10 scenarios, committed
   pass-rate report).

## Week 11 Milestone — Launch

1. **Payment rail**: Stripe (test mode acceptable) gating a real boundary (e.g., free
   tier: N courses; paid: unlimited). Full flow works with test cards; webhooks
   handled.
2. **Production bar**: custom domain with HTTPS, error monitoring quiet (no unresolved
   P0/P1 in the tracker), uptime checks configured.
3. **The stranger test** (end-to-end): a fresh user can sign up → generate a course →
   drill and get graded → simulate/return for reviews → edit the course via AI → hit
   the payment gate → pay with a test card → continue. Scripted or recorded evidence.
4. **Polish pass**: mobile-usable, loading/empty/error states throughout, no console
   errors on the happy path.
5. **Final retrospective critique** (`docs/CRITIQUE.md`): the full-Gauntlet
   retrospective — what the 11 weeks built in you, with evidence; the capstone blog
   post sources from this.
